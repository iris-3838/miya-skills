# 知識・情報の組織化（Knowledge & Information Organization）— 知識グラフ

本ドキュメントは、`lis-word-dict` の Mode 6（ドメインサーベイ）により構築した知識グラフの実例。
出典: 図書館情報学用語辞典（tosyokan_dictionary.csv, 1,796エントリ, Kotobank連携）

## 基盤概念

### 知識組織体系（KOS: Knowledge Organization Systems）

```
SKOS ──── W3C
 │           ├── セマンティックウェブ ──── RDF ──── リンクトデータ(LOD)
 │           │                                    └── オープンデータ
 │           └── オントロジー(OWL)
 │
 ├── 件名標目表 ──── 件名標目 ──── ディスクリプタ
 │                    ├── 細目(副標目)
 │                    ├── 参照形（「を見よ参照」「をも見よ参照」）
 │                    └── 分類順件名標目表
 │
 ├── シソーラス ──── シソーロファセット
 │                    ├── 多言語シソーラス
 │                    └── 医学件名標目表(MeSH)
 │
 └── 分類法 ────【十進分類法系統】
                  ├── デューイ十進分類法(DDC)
                  ├── 国際十進分類法(UDC)
                  └── 日本十進分類法(NDC)
```

- **SKOS** (2009, W3C勧告): 件名標目表・シソーラス・分類法といったKOSの構造と概念を機械可読データとして表現するモデル
- **オントロジー** (OWL): コンピュータにも理解可能な形式で概念を体系化。哲学上の「存在論」に由来。SKOSはKOS用、OWLはより汎用的なオントロジー記述言語

## 分類（Classification）

分類 = 「事象の中にクラスを概念的に認識し，クラス間の関係性を認識する過程」（辞書定義）

### 2大パラダイム

| 軸 | 列挙型 (Enumerative) | 分析合成型 (Analytic-Synthetic) |
|---|---|---|
| 原理 | 全主題をあらかじめ表に用意 | ファセットごとにフォーカスを合成 |
| 長所 | 単純・直感的 | 柔軟・複合主題に対応可能 |
| 短所 | 予測不可能な主題に弱い | 複雑・学習コスト高い |
| 代表 | DDC, NDC, UDC | コロン分類法（ランガナータン） |
| 現実 | 補助表で部分的に合成（準列挙型） | — |

### DDCの思想的系譜

```
ベーコン「学問の進歩」(1605)
  └─ 記憶=歴史 / 想像=詩学 / 理性=科学
       └── ハリス → 逆ベーコン順（科学→芸術→歴史）
            └── デューイ → DDC (1876)
                 └── NDC (1929)
```

- **十進記号法**: 1桁10区分（類）→2桁100区分（綱）→3桁1000区分（目）
- DDC第17版(1965)以降：「列挙型分類表の特徴を保ちながら分析合成型へ傾斜」
- **ランガナータンのPMEST**: Personality, Matter, Energy, Space, Time の5基本カテゴリー

## 件名標目（Subject Headings）

- **事前結合型**: 「日本―歴史」のように標目作成時に細目を結合
- **参照構造**: USE（を見よ参照）/ BT・NT・RT（をも見よ参照）
- 代表的な件名標目表: LCSH（米国議会図書館）, BSH（基本件名標目表）, MeSH（医学）
- **シソーラス**: 事後結合型（検索時に組み合わせ）。MeSHは実質的にシソーラス
- **シソーロファセット** (Aitchison, 1969): シソーラス＋ファセット分類を統合した統制語彙集

## 書誌記述と目録（Bibliographic Description & Cataloging）

### 目録法の2部門

```
目録法
  ├── 記述目録法（資料の物理的特徴 + 著者/タイトルAP）
  └── 主題目録法（分類記号付与 + 件名標目付与）
```

### 標準化の流れ

```
パリ原則(1961)
  └── ISBD (IFLA)
       ├── ISBD(G)一般枠組み(1977)
       ├── 資料別ISBD → 統合版(2011)
       └── 区切り記号法
            └── RDA/NCR2018年版では区切り記号法を不採用

各国目録規則
  ├── RDA（FRBR/LRMベース）
  ├── NCR（日本目録規則, 2018年版でFRBR/LRM準拠）
  └── AACR2（英米目録規則第2版）
```

### MARCフォーマット

```
UNIMARC(IFLA, 1977) ←── ISO 2709
  ├── 書誌フォーマット（10機能ブロック）
  ├── 典拠用フォーマット
  └── 所蔵用フォーマット
```

### 書誌ユーティリティ

- **OCLC** (1967-): WorldCat 4.5億件超、123か国17,983機関
- **NII**（国立情報学研究所）: 日本における書誌ユーティリティ
- 分担目録作業 / コピー目録作業 / 流用目録作業

## 典拠コントロール（Authority Control）

- **典拠ファイル**: 名称典拠ファイル（個人名/団体名/会議名/統一タイトル）+ 件名典拠ファイル
- **典拠コントロール**: 統制形アクセスポイントの一貫使用を維持管理
- 機能: 検索漏れ防止 / 重複レコード抑制 / DB品質管理

## 概念モデル：FRBR → IFLA LRM

### WEMI 4実体

```
著作(Work) → 表現形(Expression) → 体現形(Manifestation) → 個別資料(Item)
（抽象的）                                         （具体的）
```

### FRBRファミリーの統合

```
FRBR (1997) ──── 書誌レコードの機能要件
  ├── FRAD ──── 典拠データの機能要件
  ├── FRSAD ──── 主題典拠データの機能要件
  └── IFLA LRM (2017)
       ├── 11実体 + 属性 + 関連（階層的継承）
       └── 5利用者タスク：発見・識別・選択・入手・探索
```

- **書誌的関連**（Tillettの7群類型化）: 翻訳・改訂・翻案・補遺・全体部分・複製 etc.

## メタデータとセマンティックウェブ

| 標準 | 概要 |
|---|---|
| ダブリンコア(DCMES) | 15基本要素 (ISO 15836-1) → 55要素 (ISO 15836-2) |
| OAI-PMH | メタデータ自動収集プロトコル（データプロバイダ／サービスプロバイダ） |
| OpenURL / リンクリゾルバ | メタデータから最適リンク先を決定する仲介システム |

### セマンティックウェブ階層 (Berners-Lee, 2001)

```
RDFモデル・シンタックス層 → RDFスキーマ層 → オントロジー層(OWL) → 推論・信頼性層
```

- **リンクトデータ**: URIでデータをリンク。LOD（Linked Open Data）として図書館目録メタデータの公開進行中

## 索引（Indexing）と索引言語

- **統制索引言語**: 分類記号 / 件名標目 / シソーラス（BT/NT/RT/USE/UF）
- **自然語索引**: 全文検索 / KWIC/KWOC
- **引用文献索引**: Science Citation Index 等。引用分析のデータとしても利用
